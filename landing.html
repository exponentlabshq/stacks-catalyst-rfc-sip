Stacks Core

Block production and speed

The need for fast, consistent transactions is table stakes for any top ecosystem. Both speed and throughput are critical areas of R&D as the ecosystem scales, building off progress since Nakamoto and sBTC.

What Core Developers are working toward:

  • Short-term: Achieve consistent <10s transaction times.
  • Long-term: Become the fastest Bitcoin L2 while prioritizing robustness, decentralization, and security.
  • Ongoing: Signer-side optimizations, cost tracker updates, VM optimizations, benchmarking WASM.

Leads: Bitcoin L2 Labs, Hiro Systems, Stacks Foundation

Clarity & Wasm

New Clarity and Wasm compilation aim to remove technical debt and lay groundwork for throughput and app performance improvements.

Leads: Hiro, Bitcoin L2 Labs, Clarity4 SIP

Stacking Improvements, Ledger Live

Simplify PoX and Stacking for higher-yield UX and stronger Stacker–signer relationships; streamline pool ops; prepare for Ledger Live integration.

Leads: Alum Labs, Zondax, BD Working Group

Miner Transaction Replay on Bitcoin Reorg

Add feature for the default miner to replay orphaned transactions after a Bitcoin reorg; testnet in Q3 2025.

"PoX 5"

Exploration of an upgraded PoX informed by stacking improvements and ecosystem needs.
